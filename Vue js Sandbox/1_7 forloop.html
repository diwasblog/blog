In v-for always use Key

In vue we have two states: Application State and DOM State.

Application States are those which are defined in the data property. Ex. List of Array in data.

Temporary DOM State: They are those which are added to data let's say from the input field dynamically. This does not directly add to the actual DOM but only remains temporary in memory.

Vue’s documentation describes, “If the order of the data items has changed, instead of moving the DOM elements to match the order of the items, Vue will patch each element in-place and make sure it reflects what should be rendered at that particular index.” Let’s dig deeper.

Vue renders lists in such a way that whenever there are dynamic changes in the UI, for example, the order of the list changes, Vue will just update the data in the DOM of each element of that list. Instead of moving HTML chunks of each element of that list, Vue will just update the current application state into the DOM, by updating the DOM element’s state.

So, if order of element is changed, and if key is not given then Vue will try to add own key on each items in temporary dom, instead of moving HTML chunks. This sometimes may take time but if we use key like any unique field, or say simple a serial number ourself then vue should not have to think about it.And Hence may improve performance.

Using Name as unique field as Key in loop

<div id="app"> <ul> <li v-for="item in shoppingItems" v-bind:key="item.name"> {{ item.name }} - {{ item.price }} </li> </ul> </div> <script> new Vue({ el: "#app", data() { return { shoppingItems: [ { name: 'apple', price: '7' }, { name: 'orange', price: '12' } ] } } }); </script>

Using Index as unique field in v-for Loop

<div id="app"> <ul> <li v-for="(item,index) in shoppingItems" v-bind:key="index"> {{ item.name }} - {{ item.price }} </li> </ul> </div> <script> new Vue({ el: "#app", data() { return { shoppingItems: [ { name: 'apple', price: '7' }, { name: 'orange', price: '12' } ] } } }); </script>
