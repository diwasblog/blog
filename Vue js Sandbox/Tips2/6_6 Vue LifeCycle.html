Actions of lifecycle methods can be broken down into four categories:

  • Creation
  • Mounting
  • Updating
  • Destruction

Creation methods are implemented using beforeCreate() and created() hooks.

beforeCreate(): data is still not reactive and events that occur during the component’s lifecycle have not been set up yet. This code will be run as soon the Vue instance exists.Nothing gets called before this. 

Created():The created() You can access reactive data and event.But cannot access DOM and template and component yet. 

 

new Vue({ data: { x: 5 }, beforeCreate: function () { // `this` points to the view model instance console.log('x is: ' + this.x) } }) // x is: undefined
new Vue({ data: { x: 5 }, created: function () { // `this` points to the view model instance console.log('x is: ' + this.x) } }) // x is: 5

beforeMount():The beforeMount() method is invoked after our template has been compiled and our virtual DOM updated by Vue. After this, the $el property is added to the Vue instance, the native DOM is updated and the mounted() method is invoked.

mounted() :mounted() method gives you access to templates and enables interaction with the DOM.

new Vue({ beforeMount: function () { // el has not been created so we cannot access dom element like div .But here template is compiled console.log(`this.$el is yet to be created`) } })
<div id="app"> <p>I'm text inside the component.</p> </div> //mounted can access dom element like div . and can access DOM element new Vue({ el: '#app', mounted: function() { console.log(this.$el.textContent) } }) // I'm text inside the component.
