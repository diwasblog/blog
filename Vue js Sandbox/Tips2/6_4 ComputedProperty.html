Computed Properties 

Computed properties can be used to do quick calculations of properties that are displayed in the view. These calculations will be cached and will only update when needed.

data: { results: [{ name: 'English', marks: 70 }, { name: 'Math', marks: 80 }, { name: 'History', marks: 90 } ], },
computed: { totalMarks: function () { let total = 0; for (let i = 0; i < this.results.length; i++) { total += parseInt(this.results[i].marks); } return total; } },
Then using this as : <span> Total marks are: {{ totalMarks }} </span>

Output

Total marks are : 240 ===========================================================================================================

Achieving Same with Function

<span> Total marks are: {{ totalMarks() }} </span> methods:{ <xmp class="prettyprint"> totalMarks: function () { let total = 0; for (let i = 0; i < this.results.length; i++) { total += parseInt(this.results[i].marks); } return total; } }

Output

Total marks are : 240
==========================================================================================

Both gives same result then whats the difference between Method and Computed?

Main difference is performance hits.Using this(Method ways) syntax, the totalMarks() method gets executed every time the page renders (ie: with every change).

If instead we had a computed property, Vue remembers the values that the computed property is dependent on (ex: In the previous example, that would be results). By doing so, Vue can calculate the values only if the dependency changes. Otherwise, the previously cached values will be returned.

Because of this, the function must be a pure function. It can’t have side-effects. The output must only be dependent on the values passed into the function.
