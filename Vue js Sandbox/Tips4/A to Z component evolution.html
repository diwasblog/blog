Local Ways of Component Declaration

Here in local component we hold a component with variable name as shown below

Ways 1

<div id="app"> <user-component></user-component> </div> <script> const user = { //Defining variable for component template: '<aside>Sidebar Hero</aside>' } new Vue({ el: '#app', components: { "user-component": user } }) </script>

Ways 2

<div id="app"> <usercomponent></usercomponent> </div> <script> const usercomponent = { //Defining variable for component template: '<aside>Sidebar Hero</aside>' } new Vue({ el: '#app', components: { usercomponent } }) </script>

Ways 3

<div id="app"> <user-component></user-component> </div> <script> const userComponent = { //Defining variable for component template: '<aside>Sidebar Hero</aside>' } new Vue({ el: '#app', components: { "user-component": userComponent } }) </script>

Ways 4

<div id="app"> <User></User> <!-- Works because here first letter capital --> </div> <script> new Vue({ el: '#app', components: { "User": user } }) </script>

Passsing value in local component

<div id="app"> <usercomponent test="hello"></usercomponent> </div> const usercomponent = { //Defining variable for component props: ["test"], template: `<aside>Sidebar Hero {{test}}</aside>` } new Vue({ el: '#app', components:{ usercomponent } })
Note one things: <usercomponent></usercomponent> // we can do this <userComponent></userComponent> // we can do this <user-component></user-component> // we can do this <UserComponent></UserComponent> //we cannot do this <User></User> //we can do this Which means component inside html parts only support camelcase.

Global Ways of Component Declaration ( using as Vue.component() )

Ways 1

<div id="app"> <user-name name="Flavio"></user-name> </div> <script> Vue.component('user-name', { // different is 'user-name' props: ['name'], template: '<p>Hi {{ name }}</p>' }) new Vue({ el: '#app' }) </script>

Ways 2

<div id="app"> <user-name name="Flavio"></user-name> </div> <script> Vue.component('UserName', { // different is userName props: ['name'], template: '<p>Hi {{ name }}</p>' }) new Vue({ el: '#app' }) </script>

You can use any of this UserName or user-name  in component name ie. Vue.component('componentname',{ ... } )

But prefered ways is UserName in JS and user-name in a template.

 

Any component created using Vue.component() is globally registered. You don't need to
assign it to a variable like show below.

Creating Static Component with no message passing

// EmployeeTable.vue <template> <div id="employee-table"> <table> <thead> <tr> <th>Employee name</th> <th>Employee email</th> </tr> </thead> <tbody> <tr> <td>Richard Hendricks</td> <td>richard@piedpiper.com</td> </tr> <tr> <td>Bertram Gilfoyle</td> <td>gilfoyle@piedpiper.com</td> </tr> <tr> <td>Dinesh Chugtai</td> <td>dinesh@piedpiper.com</td> </tr> </tbody> </table> </div> </template> <script> export default { name: 'employee-table', } </script> <style scoped></style>
<!-- Using that component --> <template> <div id="app" class="small-container"> <h1>Employees</h1> <employee-table /> </div> </template> <script> import EmployeeTable from '@/components/EmployeeTable.vue' export default { name: 'app', components: { EmployeeTable, }, } </script> <style> button { background: #009435; border: 1px solid #009435; } .small-container { max-width: 680px; } </style>
In Vue, the convention is that the filename and import will be in PascalCase, such as EmployeeTable, but when used in a template, this will translate to kebab-case, </employee-table>. This uses the proper conventions for JavaScript and HTML respectively.

Creating Dynamic Component with dynamic message passing

// EmployeeTable.vue <template> <div id="employee-table"> <table> <!-- ...thead... --> <tbody> <tr v-for="employee in employees" :key="employee.id"> <td>{{ employee.name }}</td> <td>{{ employee.email }}</td> </tr> </tbody> </table> </div> </template> <script> export default { name: 'employee-table', props: { employees: Array, }, // or props: [ 'employees' ] } </script> <style scoped></style>
<!-- Using that component --> <template> <div id="app" class="small-container"> <h1>Employees</h1> <!-- Passing props to the component --> <employee-table :employees="employeesList" /> </div> </template> <script> import EmployeeTable from '@/components/EmployeeTable.vue' export default { name: 'app', components: { EmployeeTable, }, data() { return { employeesList: [ { id: 1, name: 'Richard Hendricks', email: 'richard@piedpiper.com', }, { id: 2, name: 'Bertram Gilfoyle', email: 'gilfoyle@piedpiper.com', }, { id: 3, name: 'Dinesh Chugtai', email: 'dinesh@piedpiper.com', }, ], } }, } </script>
Prop names should always use camelCase during declaration, but kebab-case in templates Following : Javascript : camelCase Html : kebab-case

Can use ternary operator like this below:

<template> <ComponentName :colored="color == 'white' ? true : false" /> </template> <script> export default { data: function() { return { color: 'white' } }, } </script>
