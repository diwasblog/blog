1) Multiple word component name ( A )

Component names should always be multi-word, except for root App components, and built-in components provided by Vue, such as <transition> or <component>.

Why? To prevent conflicts with existing and future HTML elements, since all HTML elements are a single word like <textarea> , <checkbox>

Bad (single name)

Vue.component('todo', { // ... })

Good (multiple name)

Vue.component('todo-item', { // ... })
export default { name: 'Todo', // ... }
export default { name: 'TodoItem', // ... }

2 ) Component data must be the function

Bad ()

Vue.component('some-comp', { data: { foo: 'bar' } })

Good (multiple name)

Vue.component('some-comp', { data: function () { return { foo: 'bar' }} })
//In a .vue file export default { data: { foo: 'bar' } }
// In a .vue file export default { data () { return { foo: 'bar' } } }

3 ) Prop definitions should be as detailed as possible.

If some one document this then it will be easy

Bad

// This is only OK when prototyping props: ['status']

Good

props: { status: String }

Better

// Even better! props: { status: { type: String, required: true, validator: function (value) { return [ 'syncing', 'synced', 'version-conflict', 'error' ].indexOf(value) !== -1 } } }
