Why Componentsâ€™ Data Properties Must Be Functions ?

<div id="app"> <counter></counter> <counter></counter> <counter></counter> </div> <script> </script>

With components, the data property must be a function and not an object as we are used to.

Data is the private memory of each component where you can store any variables you need. 



Notice how all of the numbers on the buttons changed. This is because all three instances of the component share the same data object because when vue invokes the function to retrieve the object, It returns a reference to the same data object each time.All three counter have the same reference to data and if any counter component changes the value then all counter component will have same value.

When clicked on first button all three button value changes.

NOw lets return data

<div id="app"> <counter></counter> <counter></counter> <counter></counter> </div> <script> new Vue({ data:{ return { counter: 0 }; } }) </script>
Here each counter component has separate copy of data object (previous each counter component has same data object) So on clicking on first button has no affect on the second and third button
