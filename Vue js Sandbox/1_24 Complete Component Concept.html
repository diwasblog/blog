Component name best way is having multiple word like app-header instead of header

Component names should always be multi-word, except for root App components, and built-in components provided by Vue, such as <transition> or <component>.

Why? To prevent conflicts with existing and future HTML elements, since all HTML elements are a single word like <textarea> , <checkbox>

Here, "This Change Title in Header" is passed/emit from Header.vue ie. child component to Parent Component ie. App.vue

Slot in Vue

Slot:
Slots are reserved space offered by vue to display HTML content passed down from one component to another.

Two Types:

  • unnamed(default) slot
  • named slot.

When to Use Slot?
- To pass down HTML elements from one component to another.

  • With props, vue allows us to pass strings, objects, arrays, and functions from parent to child component.
  • You can pass HTML elements using props but is vulnerable to CSS attack so, the slot is used which sends an insecure way.
