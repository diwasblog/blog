Software engineering is an engineering branch associated with the development of software products using well-defined scientific principles, methods, and procedures. The outcome of software engineering is an efficient and reliable software product.

 

The objective of learning software engineering

  • has an appreciation of software lifecycle models and their use in project planning and control.
  • be is familiar with different categories of software metrics.
  • be able to use techniques for the scheduling and control of large projects.
  • understand and appreciate some of the software estimation methods.
  • be able to carry out requirements and interface analysis for a software development project.
     
  • be able to construct and validate a software specification.
  • be able to use functional, OO, and formal methods for specifying software system.
  • be able to use functional, OO, and other approaches to design a software system.
  • be aware of methods for the specification of real-time systems.
  • be familiar with techniques and tools to support configuration management.
  • be able to carry out validation and testing of software
  • understand the roles and responsibilities of the members of a project team.

Process Measurement

Every company develop software and if they want to measure the capability of the 

process then they use Capability Maturity Model (CMM) model.Understanding current

model they can upgrade themselves to next higher level of maturity.

This model is developed and is promoted by Software Engineering Institute (SEI),

a research and development center sponsored by US Department of Defense(DoD)

It is not a software process Model. It is used as a benchmark to measure maturity 

of an organization's software process.

This model has 5 level of Maturity:

LEVEL 1 : Initial :

  • Project work is performed informally.
  • A software development organization at this level is characterized by adhoc activities (organization is not planned in advance(or donot think about future).

Level 2 :Repeatable

  • Work is planned and track.
  • Inherit level 1 good characteristics.
  • This level of software development organization has a basic and consistent project management (management is done here) processes to track cost, schedule, and functionality.
  • The process is in place to repeat the earlier successes on projects with similar applications.

Level: Defined

Inherit Level 1 and Level 2 good characteristics.

Work is well defined.

At this level the software process for both management and engineering activities is defined and documented like SRS.

 

Level 4 : Managed

Work is quantitatively controlled. (How much time does it takes by developing  and testing each process is tracked quantitatively ie. recorded in number like in 5 days this module will be completed)

  • Software Quality Management:Quality  of  Software maintenance and software process  is majored.
  • Quantitative Process Management: Prediction using statistical and other quantitative techniques.

Level 5 : Optimizing

Work is based upon continous improvement.

  • Process Change Management: Each phase of software development is continuously improving process. and making more flexible.
  • Technology Change Management:
  • Defect Prevention: Prevent before happening.

Software Metrics:

It is a measure of software characteristices where are quantifiable or countable.

It is important for measuring 

  • Software performance
  • Measuring productivity
  • Planning work item
  • Debugging
  • Estimation Cost.

Characteristics of software metrics:

1) Simple and computable

2) Consistent and unambiguous.

3) Use consistent units of measurement

4) Independent of programming language.

 

Classification of Software Metrics:

Good managers measures process applied to the product 

1) Product Metrics: Describes the characteristics of the product/software. Eg.

  • Size of project,
  • Complexity for building product,
  • Design features of product (coupling , cohesion,complexity
  • performance of product,quality of the product ,
  • relaiblitiy of product,
  • functionality of product

2) Process Metrics: These characteristics can be used to improve the development and maintenance.

  • How much Effort required in the process.
  • How much Time taken to build / produce the product.
  • What are the effect of using development techniques and tools.
  • What defects are found during testing.
  • Productivity
  • Quality
  • Efficiency
  • Failure

3) Project Metrics

It describe the project characteristics and execution.

  • How many developers required building the project.
  • How many staff required for building project
  • Cost building project
  • Time required
  • How much productivity is obtained.

 

By this metrics, a product manager can determine how much efficient software is developed, how much afford requirement.

So overall performance of software can be determined with the help of these 3 metrics.
