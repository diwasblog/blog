ClientSide Render : Modern Websites and application

SPA(Single Page Apps) -each page is rendered on the client , not on the server.


Mostly in vue, react and angular application. Has backend and frontend domain.

  
  

<html>
<head>
<body>
<div id="app">  </div>
  
</body>
</head>
</html>

When the browser makes the initial request to the server, the server returns the index.html HTML file which looks like up.Then it will load required js,css present in this file.


After the initial HTML and CSS loads, the browser requests the rest of the application which comes in a JS file. And then we will get JSON that then it is loaded in virtual dom then binded to id = "app" then only fully page can be seen.Chunk of js file is loaded on demanded. The content will be viewable and intractable once the JS file has been parsed.

What is pre-rendering?

Every pre-rendered page displays a skeleton template while the data waits and does ajax request and added to dom.Once the page is fetched, internal routing is done dynamically to take advantage of a client-side rendered website.

Universal Web App(isomorphic app)

Universal Web Apps combine the best of both Client Side Rendering and Server Side Rendering. In a Universal Web App, the initial render will be done in the server, and once the page loads client-side rendering will take over. This makes sure that we have good SEO, a quick first render and speed when browsing in the app.

  • Nuxt.js - A framework to build Universal Web Apps with Vue.js
