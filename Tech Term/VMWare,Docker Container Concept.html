Before Virtual Machine

We used to have bare hardware(containing multiple cpu, large capacity RAM) and on that server we install like Windows Server or Ubuntu Server.

Suppose you have installed Windows Server in bare hardware and host .net application there. Then suppose you need to host a python project, and you cannot host this project in Windows Machine. So, what we do is : 

Create Two large Bare Machine

1)Bare Hardware with Windows Server ( ==> Host .net application)     

2)Bare Hardware with Ubuntu Server ( ==> Host php, pyhton application)

So , two bare pieces of hardware are required.

So what is the problem here is: We need two bare hardware which is more expensive.

To solve this problem, Virtual Machine, Hypersior Came.

So, instead of using two bare hardware, we use one.

Method1: In that hardware we installed Hypervisor and on that, we installed multiple Virtual Machine ie. multiple OS.Which is called Type 1 Hypervisor.

Method 2: In that hardware we installed one OS called Host OS , and on that Hypervisor is installed and on that we installed multiple Virtual Machine OS. This is called Type 2 Hypervisor.

 

Role of Hypervisor:

You can create One Windows OS VM where you host .net Application, One Next VM installed Ubuntu Server and on that host PHP, python application. So, one hardware ==> multiple software.

Then Container and Docker Came

What happens here.

We have bare hardware, then we installed one OS, say Ubuntu Server  and on that Docker is installed (installed like any other application in Linux and this process keep running) and on docker we installed Operating System ( which is called container)

Both Seems look similar ,

  • ie The container has Operating System or any application
  • Virtual Machine has an Operating System or any application.

Different is: But Docker Engine is lighter and Faster than Hypervisor. So, a Virtual Machine takes more time to boot/start than Container.
