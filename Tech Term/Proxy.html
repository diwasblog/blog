In forward proxy, the server does not know who the is client. In Reverse Proxy, Client doesnot know which server they are using for getting data

Used for Anonymity: Here, client becomes annonymous to server.

User for Caching : Here, proxy server caches static files and when client request for the same resources then, proxy server provides its cache data.

Used for Blocking Unwanted Sites: In nepal porn sites are banned.So, here ,in ISP proxy  server they banned certain websites like pornhub.com and when client request they will get some page like not found or not allowed page set by ISP.

Used of GEOFancing: Allowing only certain user under certain GEO location can access the data.

Used for Load Balancing: Here , reverse proxy helps to balance the load.Suppose server 1 has high traffic, then reverse proxy send it to Server2 and soon. This will allow you to eliminate downtime and increase productivity 

Amazon, Google uses these techniques.If you search from nepal then you will get google.com.np version of google search and soon 

Used for Caching:Same as Forward PRoxy

Used of Isolating Internal Traffic:Here internal traffic between reverse proxy and Servers are isolated, no port and ip,machines are known.Also Most user credentials are just stored on the file transfer servers themselves. So if your file transfer servers are placed on your DMZ, a highly-motivated attacker may easily get a hold of them. By moving your servers into your internal network and deploying a reverse proxy to control access, you can provide better security to those credentials and, consequently, to the data they protect. 

Used for logging: Reverse proxy handles logging information like traffic, failure.
