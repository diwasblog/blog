What is Modular Application Development?

In modular Application Development, we create multiple independent modules of software and later combine all to make a large application. Here each module is independent and can be reused in other applications as well. Finally, all the modules are smushed/combined in one application and deployed in one machine. Eg. Installed Photoshop, each has a separate copy on their system.

Then came web application, here follow the same modular techniques but hosted on a web server so that multiple users use the same application from the server. Eg. Web Photoshop

These both desktop and web applications follow monolithic architecture. Ie .Single Store Architecture. 

Biggest Disadvantage of Monolithic Architecture is

1) " Bigger the application, difficult to deploy it."

Suppose you have made an application and if say in module 4 small changes has to be made then we have to build whole system and have to check whether  module4 has effect on other module or not. And if no negative effect, then publish the application.

2) Scalability

Say e-commerce sites, we know that eCommerce sites has large spikes on a certain day.

If you have eCommerce sites then if traffic has raised on your sites then you will scale the whole application server has to scale. 

Suppose say if only in certain modules has much traffic then also we have to scale whole application. So, lots of money may be required to duplicate the instances of whole application.

3) Single Point of Failure

If one modules failed then will not work another module as well because we have to publish on single system

Microservices Application

Then came microservices architecture, here we have self-working modules(mini-application) and each of these modules is hosted on a separate server. So, if we have more traffic on particular module then we can just scale its server. And shouldnot have to touch other servers. So, it minimizes the cost.

 

Here each mini-application can communicate with each other via APIs and together works as a bigger application.

 

So this mini-application (self-completed application) which are hosted on different servers and can communicate with each other via APIs.So this mini-application are called as microservices.

Advantages of Microservices:

1) Deployment Flexibility

Different developers can work on each mini-application and can be flexible to store in different servers. These mini-application can communicate via APIs(Language Independent) and makes a complete system.

2)Language Flexibility

This mini-application can be built in any language and these applications can communicate with Apis easily. And finally forms a big application.

3)Can Be Scaled Separately

Each mini application can be scaled separately

Disadvantages of Microservices:

1) Deployment/Architecture Complexity

Can to make an architecture to host in separate server and communicate each other efficiently.
