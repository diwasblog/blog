When you as developer build your application and from your perspective, you check the project and is working while and then now you will give the project(via zip or anything) to tester then tester will load the project and tester will find that project is not working in their environment because there may be dependency issue. Suppose you are using windows 10 and tester is using windows xp then there may be a dependency issue. Suppose say you have .net framework 4.5 installed in your machine and tester just has .net framework 4.0 then there may occur problem in a testing environment.

 

The solution is VMWare or Hypervisor 

In VMWare or Virtual Machine or Hypervisor, you can install OS, software, and application. 

To solve this problem developers create an image of working OS (containing working project along with dependencies) and then gives the entire image of OS then the tester will load the image then test it. Here, tester has not to worry anything because the developer has given everything. Then tester creates test code and everything works well then they again create an image of OS and give it to operation team. Then the operation team checks it and if all OK then they install the image of OS with software in the production server.

 

Suppose say you have 10 application then may have 10 different types of image of OS then these 10 virtual OS is installed on the main server (say Linux OS). But the problem with virtual OS is :

  • They take hardware space (HDD, RAM, Processor) 

 

Again to reduce resource usage: Docker came into the picture
