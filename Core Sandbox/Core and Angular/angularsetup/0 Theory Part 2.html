TrackBy

Track By clause allows you to specify your own key to identify objects.

Angular uses the object identity to compare the elements in the collection to the DOM nodes. Hence when you add an item or remove an item, the Angular will track it and update only the modified items in the DOM. It does not render the entire list.

But this fails if we update the list from the back end server. That is because the retrieved objects cannot be compared with the existing objects in the list as the reference has changed. The Angular to simply remove these elements from DOM and recreates the new elements from the new data. This has a huge performance implication.

 

So, when the backend row added/ deleted then DOM will not fully reload if used trackby as we are tracking by unique id of the table as shown below.

So, here even new row is added /delete from database then also whole table willnot be refreshed by that particular row get refreshed 
    
        //app.component.html
        <tr *ngFor="let movie of movies; trackBy:trackByUniqueField>;">
            <td>{{movie.title}}
            <td>{{movie.director}}</td>
            <td>{{movie.cast}}</td>
            <td>{{movie.releaseDate}}</td>
        </tr>


        //app.component.ts


        trackByUniqueField(index, item) {
            return item.title;    //consider title as unique field
          }

          The trackBy takes a function that has two arguments: index and the current item. 
          It must return a id that uniquely identifies the item.

Track by Multiple Field

    
        //app.component.html
        <tr *ngFor="let movie of movies; trackBy:trackByUniqueMultipleField>;">
            <td>{{movie.title}}
            <td>{{movie.director}}</td>
            <td>{{movie.cast}}</td>
            <td>{{movie.releaseDate}}</td>
        </tr>


        //app.component.ts


        trackByUniqueMultipleField(index, item) {
            return item.title + item.cast;    //consider title and cast makes unique field
          }

          The trackBy takes a function that has two arguments: index and the current item. 
          It must return a id that uniquely identifies the item.
