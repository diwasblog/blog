1. Interpolation {{}}
    Data is bind from component to View
  2. Property Binding [ ] or "bind-"
    Data is bind from component to the property of an HTML control in the view like
  3. Event Binding  (click) or "on- "
    The DOM Events are bind from View to a method in the Component
  4. Two-way Binding/Model Binding  [(ngModel)] or "bindon-"
    The data flow in both directions from view to component or from component to view
  • {{ templateExpression }}  ====> The content inside the double braces is called Template Expression
  •  {{ }} is called interpolation. 
  • Interpolation is one-way binding: when component changes then view change, but when view change component value does not change

Attribute binding

Sometimes there is no HTML element property to bind to. The examples are aria (accessibility) Attributes & SVG. In such cases, you can make use of attribute binding

The attribute syntax starts with attr followed by a dot and then the name of the attribute as shown below

Example: 
    
        <button [attr.aria-label]="closeLabel" (onclick)="closeMe()">X</button>

       paragraph changes color

        //app.component.ts
        export class AppComponent {
            isBold: boolean = true;
            fontSize: number = 30;
            isItalic: boolean = true;
        
            addStyles() {
                let styles = {
                    'font-weight': this.isBold ? 'bold' : 'normal',
                    'font-style': this.isItalic ? 'italic' : 'normal',
                    'font-size.px': this.fontSize
                };
        
                return styles;
            }
        }

Event Binding Angular

Event binding allows us to bind events such as keystroke, clicks, hover, touche, etc to a method in component.

It is one way from view to component. 

sitecomponent.html 
            
              
                <button (click)="clickMe()">Click Me</<button>             //  on-click="clickMe()"
                <p>You have clicked {{clickCount}}</p>
sitecomponent.ts 
          
            clickCount=0
            clickMe() {
              this.clickCount++;
            }

Passing Event to Function

appComponent.html 
            
              
                <input (input)="handleInput($event)">
                <p>You have entered {{value}}</p>
appComponent.ts 
          
            export class AppComponent {
            value=""

            handledInput(event) {
               this.value=event.target.value;
             }

Two Way Binding / Model Binding

Two-way data binding means that changes made to our model in the component are propagated to the view and that any changes made in the view are immediately updated in the underlying component data.

Notes

Angular uses NgModel for two way data binding.

To use NgModel we need to import FormsModule and add it to imports attribute of @NgModule in our module file.

Using NgModel with Text Field

appComponent.html 
            
                <input [(ngModel)] ="myMsg"/> 
                {{myMsg}}


                // Or 
                <input bindon-ngModel ="myMsg"/> 
                //OR 
                <input [value] ="myMsg" (input)="myMsg=$event.target.value"/> 
                //OR
                <input [ngModel] ="myMsg" (ngModelChange) ="myMsg=$event"/>
