Cross Site Scripting Attack(XSS Attack)

Cross-site Scripting (XSS) is an attack in which malicious scripts are injected via input fields this attack is most common and allows an attacker to steal credentials and valuable data that can lead to a big security breach.

In this attack, the attacker visits a website and tries to execute a malicious script in the form comment box. Now if the website has not checked for Malicious code then the code can get executed on the server and saved into the database.

 

Let's try to understand the same using an example. Below is a simple Employee form which we are trying to save data. Now in the text box, I am trying to execute some malicious code using javascript using the SCRIPT tag. But if we try to submit the same MVC throws an error that something bad is happening.

 

Note: Asp.Net itself prevent XSS and do not allow to submit of the form. This is good.:)

Programming sites should allow to post script as well in the comment so how to allow those?

 

So we have four things via which we can allow scripts to be posted.

Solution: -

  1. [ValidateInput(false)]
  2. [AllowHtml]
  3. [RegularExpressionAttribute]
  4. AntiXSS Library

 

[ValidateInput] is an attribute which can be applied on Controller or Action Method on which we want the script to go through.

AllowHtml: [AllowHtml]attributes that is applied to Model properties

Using Regular Expression in Model Properties

AntiXSS :Using Anti XSS library.
