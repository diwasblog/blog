Now using Attribute Routing 

      
        [Route("api/[controller]")]
        public class ValuesController : Controller
        {
            [HttpGet("{id}")]
            public string Get(int id, string foo, string bar)
            {
                return id + " " + foo + " " + bar;
            }
        }
    
      
    
      
        In this case, the action takes 3 parameters – id, foo and bar. 
        However, only id is mandatory – because it’s part of the route,
         the latter ones are optional. This means that all 4 of the following 
         URLs would be valid and lead to our action:
        
        GET api/values/5
        GET api/values/5?foo=a
        GET api/values/5?bar=b
        GET api/values/5?foo=a&bar=b
        GET api/values?id=5&foo=a&bar=b   // here we donot have Optional parameter 
        
        But Get api/values will not work because here int id is not nullable . So must request in Url
