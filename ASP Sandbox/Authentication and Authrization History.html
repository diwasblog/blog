Authentication and Authorization History

Asp.Net framework provides authentication in 3 different ways :


1) Forms Authentication
2) Windows Authentication
3)Passport Authentication

In Normal Forms Authentication, the asp.net framework does not create any table related to the user and we have to create ourself table. And Manage own authentication and authorization. Which is called Custom Form Authentication.

 

Classic Membership

Then the next came the provider-based model, which is called Asp.Net Membership which simplified the process of Authentication and Authorization to store user profile information. This is called as Classic Membership. It was introduced in 2005 NOV, with ASP.Net 2.0

Features of Classic Membership

  • Great at that time.
  • It is mainly designed for SQL Server only to store data for users.
  • It generally involved username/password-based registration.
  • Here required full SQL server for default cases. they depend on stored procedure supports, and they rely on SQL server
    cache dependencies.
  • It doesn't work with SQL Server family-like SQL Azure, SQL Server, SQL Server CE, SQL Server Express, LocalDb
  • SQL server-based providers do not work well on SQL Server CE, SQL Azure and It is not simple to extend default functionality and add custom user fields,or to move data to other kinds of database. The membership database schema is designed for SQL Server only. It is hard to move data to other data sources, especially to non-relational databases.
  • It has generated its own table, store procedure, and view. So we should not have to do our effort in creating those things and uses Form-Based Authentication in C#.

Universal Provider:

  • Came in May 2012  AD. It now works well on SQL Server family like now work well on SQL Azure, SQL Server, SQL Server CE, SQL Server Express, LocalDb.
  • It simplified Classic Membership by removing Store Procedure and Views. and also simplified table structure.

Simple Membership

  • It was introduced in Aug 2012 and WebMatrix.
  • Advantages of SimpleMembership over classic Membership is :in simple membership, it is easier to customize user profiles. 
  • Has all features of the above two methods, but it doesn't work with other databases like Mysql and No SQL. Also, OWIN support doesn't exists.
  • It doesn't works with external authentication providers like Facebook, Gmail, Twitter etc.

All of these above only works with Form Authentication and IIS Integrated Authentication, SQL Server Environment, Dependency on System.Web.

 

As the internet is evolving day by day and people are comfortable with social media identity to log in to various services instead of managing separate credentials for every service. So Microsoft developed ASp.Net Identity System.

Asp.Net Identity

Asp.Net Identity is newly designed and built from scratch addressing all of above problems.So, It is designed to make it the next single Identity system to work accross systems like MVC, WebForms, WebPages(WebMatrix), Web API, SignalR, SmartPhone,Hybrid System, Social Credential etc.

Has two version : Asp.Net Identity V1 (2013 ) , Asp.Net Identity V2 (2014)

Features:

  • Fully Support OWIN
  • Works well with External Login Providers or OAuth Providers through middleware components.
  • Works across applications, Phone, MVC, WebForms, Hybrid-app, etc.
  • Supports Role based and more powerful Claims Based Identity system.
  • Works from MVC 5
  • Provides persistence with Entity Framework Code-First, thus it will support all databases that supports EF.

In visual studio 2013, we donot have form authentication but instead we have Individual Authentication for ASp.Net MVC 5 Project .So this Individual Authentication is ASP.NET IDENTITY. Asp.Net Identity is part of all Web project template from Visual Studio 2013.

  • What is OWIN (Open Web Interface for .Net. ) ?
    ASP.NET was always tightly integrated with IIS.
  • It was quite difficult to run .NET web apps outside IIS.
  • The reason for OWIN was to provide a modular and extensible HTTP server pipeline that would be able to run without IIS dependencies.
  • This allows e.g. self-hosting ASP.NET Web API or SignalR applications in Windows Service and other interesting scenarios.OWIN Security is a set of middlewares which handle the authentication.
