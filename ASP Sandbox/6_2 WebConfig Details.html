Web Config File:

Used to specify the settings for every ASP.NET application on a machine, a particular ASP.NET application, the ASP.NET pages in a particular directory, or a single ASP.NET page.

 

  • WebConfig naming convention: camelCase

  • case-sensitive: Yes

  • If you make any changes in web config file then you should not have to build or restart application and web services,it will auto-detect changes.

     

    Term

    Explanation
    trace This section contains configuration settings for page and application tracing. 
    globalization This section specifies the character encoding to use with both requests and responses.
    httpRunTime This section specifies the maximum amount of time that a page will execute without timing out, the maximum size of a request and whether fully qualified URLs should be used for client redirects.
    customErrors This section specifies how error information should be displayed. 
    authorization This section specifies the users and roles authorized to access files. 
    machineKey This section is used for sharing a standard encryption key across machines in a Web farm. 
    httpHandlers This section associates a particular HTTP handler with a particular page path and request verb. For more information
    httpModules This section lists the modules that are involved in every page request. 
    webControls This section specifies the location of the client-side script library used with Web controls, such as the validation controls
