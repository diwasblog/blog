MVC API vs WebAPI

MVC API  WebAPI
 Used to create web applications that returns both views and data Used to create webapplicatoin that returns only data not view.
MVC Application project type adds web specific things like Scripts,Content for CSS and Javascript. WepApi Application doesnot have Scripts and Content as it doesnot have view.
Web API helps to build REST-ful services over the .NET Framework and it also support content-negotiation(it's about deciding the best response format data that could be acceptable by the client. it could be JSON,XML,ATOM or other formatted data), self hosting which are not in MVC. Doesnot support content-negotiation . It support JSON type data  or XML type data format only and we return this result via .JSONRESULT

Model + View===>Html  (MVC Project)

Model+ContentNegotiation ===>Result (API)

Web API Self Hosting

You can mix Web API and MVC controller in a single project to handle advanced AJAX requests which may return data in JSON, XML or any others format and building a full blown HTTP service. Typically, this will be called Web API self hosting.

When you have mixed MVC and Web API controller and you want to implement the authorization then you have to create two filters one for MVC and another for Web API since boths are different.
