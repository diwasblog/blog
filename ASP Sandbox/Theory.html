Roslyn- Open Compiler vs Traditional Closed Compiler

Traditional compiler were black boxes -- source code goes in one end, magic happens in the compiler(in middle), and then object files or assemblies come to other ends. All the main things are done by the compiler and we don't know what is happening inside. The only employee of the Microsoft building compiler knows all about it. This is one of the downside as the code of compiler is not public.

So, Microsoft introduced an open compiler called Roslyn for everyone and is hosted on GitHub and is used from visual studio 2015. This allows you to see the way it works and in case of an error — report it to the developers. So, the developer if found any error then he/she can looks on the compiler code on Github and suggestions for improvement to Microsoft.
