Worker

A worker is a script that runs on a thread separate to the browser’s main thread.If too many activities in the Main thread will slow down your sites. All the script tag that we use in webpages works in the Main Thread.

Web Workers, Service Workers, and worklets are all scripts that run on a separate thread. They provides possibility to write multi-threaded Javascript. They do not block UI and user interaction.Workers utilize thread-like message passing to achieve parallelism.

Web workers:

It can load any heavy processing from the main thread. Web workers are created using the Web Workers API. After creating a dedicated Javascript file for our worker, we can add it as a new Worker.

Web Workers run in an isolated thread. As a result, the code that they execute needs to be contained in a separate file.

Service Workers:

It is designed to be fully async; as a consequence, APIs such as synchronous XHR and localStorage can't be used inside a service worker.

It acts as a proxy between browser and network and works independently from an app.

Required HTTPs and allow http only on localhost.

Converting standard websites into PWA we add a couple of layers. First is : Service Worker sw.js is client-side which acts like client-side proxy object. Its key role is to cache content for that it uses caches api. It provides easy client-side storage for any resource on the page.

PWA is just an enhancement of general websites and if PWA is not active then it works like general sites.

Progressive Web Apps Requirement

  1.  Having a website with a cache is great.
  2.  The website has to be designed as Mobile App like Behaviours, so it should be responsive, fast, and runs smoothly in any device.
  3. While using mobile, data usage is important and use the network only when it has to.
  4.  Has to work offline to some degree.
