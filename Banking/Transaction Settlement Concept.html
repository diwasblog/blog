When should I void a transaction and when should I refund the transaction?

Suppose you are buying goods from the supermarket, then you will give him Credit Card / Debit Card for payment.

Void Transaction

  • When Credit/Debit Card is inserted into POS machine, then POS authenticate the user and check whether the user has sufficient balance or not, if has sufficient balance, the required amount is withdrawn from the User Account.
  • When the User cannot use that transaction amount. 
  • And this will get stored in the POS system say in the vault (digital form) and will not go to the merchant account in bank at that time. At the END of DAY (EOD) or at night, all transactions are batched up for settlement(sync with the bank) by supermarket staff. And at that point, the funds are transferred to a merchant bank account.  

 

  • If the transaction is not settled (or say money is in the POS vault ) then you can void the transaction. 
  • When the transaction is void by the merchant then the User will get his money back with no transaction fee because that money was just in the vault.

 

Refund Transaction

If the transaction is settled( ie. say money is sync with merchant Account in the bank) then you can refund the transaction.

In this process, the User has to wait 4-5 business days for getting his money back. This process is called Refund Transaction.

 

Conclusion:

So, if the transaction is in the unsettled stage then we can void the transaction.

If the transaction is already settled then we can refund the transaction.
