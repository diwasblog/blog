Merge Type

        
           1. Fast Forward
           2. 3 Way Merge

Fast Forward

Note: Git head will always point to the current working Branch.

After the merge, Master will move to devbranch and head will point to Master.

    
git init
// ... make changes 
git commit -m "added file.text C1"
// ... make changes
git commit -m "added file2.text c2"
git checkout -b devBranch
// ... make changes
git commit -m "make changes for c3 commit"
// ... make change
git commit -m "make changes for c4 commit"
// ... make changes
git commit -m "make changes for c5 commit"
git checkout master     
git merge devDiwas

3 Way Merge Concept

From master C2 commit, devBranch is created and devBranch Moves on also at the same time master also goes on.

So, to get back master data to devBranch, we pull the master branch and get in devBranch.

    
git init
// ... make changes 
git commit -m "added file.text C1"
// ... make changes
git commit -m "added file2.text c2"
git checkout -b devBranch
// ... make changes
git commit -m "make changes for c3 commit"
// ... make change
git commit -m "make changes for c4 commit"
// ... make changes
git commit -m "make changes for c5 commit"
git checkout master     
git merge devBranch  


//Some works on Master Branch 
// .. make changes on master branch
git commit -m "make changes on C6 commit"
// ... make changes 
git commit -m "make changes and commit C7"


//Then you want to get master Data to DevBranch then you have to pull master first , because master is ahead  from parent of devBranch 
git pull origin master  // you may get conflict

//Or
git checkout master
git merge devBranch
