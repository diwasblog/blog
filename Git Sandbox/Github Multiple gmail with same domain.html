Windows Credential Manager Store password based on domain and Domain for both git:https://privateuser@github.com and git:https://workuser@github.com is the same. Both have domain github.com.

 

Solution for storing multiple Github Account

1) First Delete Existing Github Account From Windows Credential Manager

Go to Windows Credential Manager, open the Windows Credentials tab, locate git:https://github.com, open the entry, and click Remove.

Here we have git:https://github.com , just click on red rectangle and click on remove link. 

Why So?

Because when you try to store two github repos using two credentials. Say : One Github repo with poudel.diwas777@gmail.com and another has Github repo with poudel.diwas789@gmail.com then Say your first github repo in your computer is set to poudel.diwas777@gmail.com and it will set in git:https://github.com by default and when you try to push second github with poudel.diwas789@gmail.com then you may get  error like this:

fatal: Authentication failed for https://www.github.com/name/repo.git

So , We are deleting default : git:https://github.com

2 )To recognize two separate accounts from the same URL for Credential Manager

Open Command Prompt and type: 
    
        git config --global credential.github.com.useHttpPath true

For getting windows credential manager to recognize two separate accounts from the same URL.

3)Reauthenticate

Just work on your project. Whenever GitHub asks you for your credentials, just type your username and password. Windows will store them and you never ever have to provide them again for this repo.
