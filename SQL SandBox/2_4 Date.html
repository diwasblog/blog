SQL Server comes with the following data types for storing a date or a date/time value in the database:

  • DATE - format YYYY-MM-DD
  • DATETIME - format: YYYY-MM-DD HH:MI:SS
  • SMALLDATETIME - format: YYYY-MM-DD HH:MI:SS
  • TIMESTAMP - format: a unique number

You can also format the date or time without dividing characters, as well as concatenate the date and time string:

Sample statement Output
select replace(convert(varchar, getdate(),101),'/','') 12302006
select replace(convert(varchar, getdate(),101),'/','') + replace(convert(varchar, getdate(),108),':','') 12302006004426

CONVERT vs CAST

CAST CONVERT
Both convert data from one form to another.  

CAST is ANSI-standard function

Means it will work when you port your T-SQL queries to a non-Microsoft database also.

 CONVERT, however, is a function specific to T-SQL.

SELECT CAST(25.65 AS int);

Output:25

SELECT CONVERT(int,25.65);

OUtput:25

 
 

SELECT CAST(25.65 AS int);

Output: 25




Always convert  will transforms into varchar(10).

SELECT CONVERT(VARCHAR(10), date_of_birth, 102)

Suppose you want to convert that date string into date then use both CAST and CONVERT as below

SELECT CAST(CONVERT(VARCHAR(10), date_of_birth, 102) AS DATETIME
