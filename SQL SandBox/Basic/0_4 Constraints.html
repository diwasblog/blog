Constraints

Constraints are the rules that we can apply on the type of data in a table.

 Available constraints in SQL are:

NOT NULL:

This constraint tells that we cannot store a null value in a column. 

      
       CREATE TABLE Student
(
ID int(6) NOT NULL,
NAME varchar(10) NOT NULL,
ADDRESS varchar(20)
);
      
    
his Query creates a table Student with the fields ID and NAME as NOT NULL.



UNIQUE
This constraint helps to uniquely identify each row in the table. i.e. for a particular column, all the rows should have unique values.
      
       CREATE TABLE Student
(
ID int(6) NOT NULL UNIQUE,
NAME varchar(10),
ADDRESS varchar(20)
);



So , Id column of Student table must be unique.(means no two Id are same)


PRIMARY KEY
Primary Key is a field which uniquely identifies each row in the table.
Primary Key=Not Null +Unique
A table can have only one field as primary key

      
       
CREATE TABLE Student
(
ID int(6) NOT NULL UNIQUE,
NAME varchar(10),
ADDRESS varchar(20),
PRIMARY KEY(ID)
);

FOREIGN KEY

Foreign Key is a field in a table which uniquely identifies each row of a another table. That is, this field points to primary key of another table. This usually creates a kind of link between the tables.

CHECK

Below query creates a table Student and specifies the condition for the field AGE as (AGE >= 18 ). That is, the user will not be allowed to enter any record in the table with AGE < 18.

  
CREATE OR ALTER TABLE Student
(
ID int(6) NOT NULL,
NAME varchar(10) NOT NULL,
AGE int NOT NULL CHECK (AGE >= 18)
);

DEFAULT
This constraint is used to provide a default value for the fields.

Below query will create a table named Student and specify the default value for the field AGE as 18.

  
CREATE OR ALTER TABLE Student
(
ID int(6) NOT NULL,
NAME varchar(10) NOT NULL,
AGE int DEFAULT 18
);
