Delete vs Truncate

Delete Truncate
We can delete all data or condition data with Delete Statement Truncate removes all data from table 
Deleted data can be roll back Truncated data cannot be roll back 
                
                    Select * from student;
                    Delete from student s where id =5;
                    rollback;
                    Select * from Student; // You will see deleted data to be there
                
                
                
                    Select * from student;
                    truncate table student ;
                    rollback;
                    Select * from Student; // You willnot get old data
DELETE TABLE statements delete rows one at a time, logging each row in the transaction log, as well as maintaining log sequence number (LSN) information. TRUNCATE TABLE is a statement that quickly deletes all records in a table by deallocating the data pages used by the table. This reduces the resource overhead of logging the deletions, as well as the number of locks acquired; however, it bypasses the transaction log, and the only record of the truncation in the transaction logs is the page deallocation.
Has restore features and log maintain so slower Donot maintain log and has no restore features so is faster than delete
