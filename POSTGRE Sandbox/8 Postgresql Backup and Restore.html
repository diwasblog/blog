Tar vs Zip vs gzip

.tar == uncompressed archive file
.zip == (usually) compressed archive file
.gz == file (archive or not) compressed using gzip tool.

.tar.gz == So, in order to create a compressed archive, you have first to create an archive using the tar utility here we get .tar file after that you will compress that archive. Resulting .tar.gz file. Also called as.  .tgz 

 

Tar File

One key thing to remember is a plain tar file is just an archive whose data are not compressed. In other words, if you tar 100 files of 50kB, you will end up with an archive whose size will be around 5000kB. The only gain you can expect using tar alone would be by avoiding the space wasted by the file system as most of them allocate space at some granularity (for example, on my system, a one byte long file uses 4kB of  disk space, 1000 of them will use 4MB but the corresponding tar archive “only” 1MB).

Conclusion:

If File system allocate 4kb each file, but actually 1 file each has cost of 1Kb then suppose we have 5 file then with tar file size leads to (5*4) Kb = 20 Kb but tar leads to (5*1)Kb = 5Kb only

Backup Postgresql particular table using command

Syntax: 

pg_dump -h localhost -p portnumber -U username -F c -b -v -f "location where you want to backup" -t table1 -t table2 database_name

Example

pg_dump -h localhost -p 5431 -U postgres -F c -b -v -f "E:/employee.backup" -t public.categories -t public.customers northwind2
