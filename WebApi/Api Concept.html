GET, PUT and DELETE are idempotent means making same request donot effect the resource.

Making as many GET request has no affect on the server.

We  can make DELETE request as many times as we want as only the first request has affect on the server and other donot.

ie .delete ; api/student/10 ==> this will delete the record for the first time called and on next call to this url then no affect on server because record with id 10 will not exist.

When we make PUT request then only first request has affect on the server and when request PUT request again then has no affect on the server.

ie  api/student/update/10  ==>While updating we have ID and when we make first request then we  make second request for  update then also we get the updated to 

But POST is non idempotent because each times you run POST then new resource will be created.And on each time you request will affect the server so API creater have created POST as separate.
