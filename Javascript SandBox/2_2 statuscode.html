2xx success status 

200  Ok

General success status

201 Content

Created some content and sent back that code to the browser. 201 states that content is created.

You can return 200 OK but browser or person consuming API might get confused because it says nothing.

204 No Content

When making API call for delete then if content deleted then send 204 success. No content to return when delete. You can return 200 but browser or person consuming API might get confused because it says nothing.

3xx Redirect

301 Move Permanently

304 Not Modified 

Get cache api. If server response with 304 not modified then client /browser will have to load from its cache because nothing is changed.

4xx Client Side Error

 

400 Bad Request

General bad request. We do not know what is bad but bad actually happens.

Suppose API needs username and email as a parameter but the client sends the only username then it says 400 Bad requests.

401 unauthorized ===> Authentication Error

You must be logged in or API key but you have not sent.

I means I donot know who you are.

Trying to Accessing resource where you donot have any right. Deleting file where you donot have right

403 forfidden

They gets logged or api key but say this user is general user and if trying to access admin panel then it says forbidden.

I know who you are but you donot have right to access this url/api.

404 not found

Not found api or pages which is being requested.

5xx error

500 Error

This is server error. Suppose you are requesting for api then if your donot response, or has server error or error in code or server down then it return 500 to client.
