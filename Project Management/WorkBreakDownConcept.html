WorkBreakDown

WBS (Work BreakDown Structure )

It is a process of breaking a large and complex project process in simpler and manageable hierarchical tasks.

Project can be any construction or software project

  • To ensure all the work required to meet the project objectives is identified, defined and assigned. (प्रोजेक्टमा चाइने सबै कुरा हरु खोज्नु and मान्छेलाई assigned garne like do this work in this time and soon).
  • To estimate the time and cost that is taken by the various elements of the project. 

Some Characteristics:

  • Definable: can be described and easily understood by project participants.
  • Manageable: a meaningful unit of work where specific responsibility and authority can be assigned to a responsible individual.
  • Estimateable : Duration to complete a part can be estimated.
  • Measurable: Can be used to measure progress.
  • Adaptable: Addition and removal of a part can be easily adopted.

To perform this WBS there are some steps and they are following top-down approaches.

Step:

1) Identify all the major elements of the project.

2) Identify supporting activities of the major elements.

3) Repeat till undivided activities are created.

Suppose we have a project, then 

1) We break down this project into 3 tasks Task 1 , Task 2 and Task 3.  (Which are major element of the project, when we combine them it should form a project) . Which are level 1 task .

 Some users may be assigned to this level to handle/provide the task to a lower level.

2) We then further divided into a subtask. For Task 2 we divided into sub-task 1 , 2 and 3 .  Which are level 2 task. When we combine all tasks on level 2 then it should form a project.

 Some users may be assigned to this level to handle/provide the task to a lower level.

3) We further divides project into work package . Which is considered as unbreakable and cannot be break or subdivided further. And this work package is assigned to the user as main to do task.

These work package are logically independent manageable blocks of work. 

When we combine all the work package then it will also form a project.

 

Note:

  • There can be numerous other level . For simplicity we are taking only 3 levels.
  • More the level , more difficult to handle the project.
